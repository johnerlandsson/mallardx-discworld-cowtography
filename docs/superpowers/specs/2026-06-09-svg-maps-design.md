# SVG Maps Implementation Design

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace PNG map images + canvas marker overlay with inline SVG maps where room nodes, connection lines, and visual state (current room, route highlights) are all CSS-driven SVG elements, enabling native zoom/pan and full Mallard theme support.

**Architecture:** A build script generates one SVG per map from the existing SQLite DB. Each SVG has four named layers: two generated (exits, rooms) and two hand-crafted (artwork, labels). Re-running the build script only replaces the generated layers, preserving all manual work. The viewer inlines SVGs into the DOM and toggles CSS classes for state — no canvas drawing.

**Tech Stack:** Node.js build script (SQLite via better-sqlite3), inline SVG, CSS custom properties, vanilla JS viewBox pan/zoom.

---

## Scope

- **World disc (map 99):** excluded — stays as PNG + canvas. No changes.
- **UU Library (map 47):** included but special — tile-grid SVG (see below), not DB-generated.
- **All other 63 maps:** SVG generated from DB room coordinates and exits.
- **Branch:** `feature/svg-maps` off current `main`.

## SVG Layer Structure

Every generated SVG contains four named `<g>` layers in this order (bottom to top):

```svg
<svg xmlns="http://www.w3.org/2000/svg"
     viewBox="0 0 {maxX} {maxY}"
     class="map-svg"
     data-map-id="{mapId}">

  <g id="layer-artwork"><!-- artwork --></g>

  <g id="layer-exits">
    <line id="edge-{sortedHashA}-{sortedHashB}" class="exit"
          x1="{x1}" y1="{y1}" x2="{x2}" y2="{y2}"/>
  </g>

  <g id="layer-rooms">
    <!-- outdoor rooms (topLevel maps) -->
    <circle id="room-{hash}" class="room outdoor" cx="{x}" cy="{y}" r="8">
      <title>{shortName}</title>
    </circle>
    <!-- indoor rooms (sub-maps) -->
    <rect id="room-{hash}" class="room indoor"
          x="{x-8}" y="{y-8}" width="16" height="16" rx="4">
      <title>{shortName}</title>
    </rect>
  </g>

  <g id="layer-labels">
    <text class="map-label" x="{x}" y="{y-12}">{shortName}</text>
  </g>

</svg>
```

**Layer responsibilities:**

| Layer | Owner | Build script behaviour |
|---|---|---|
| `layer-artwork` | Hand-crafted | Seeded empty on first run; never touched after |
| `layer-exits` | Generated | Fully replaced on every run |
| `layer-rooms` | Generated | Fully replaced on every run |
| `layer-labels` | Hand-crafted | Seeded from room names on first run; never touched after |

When the build script detects an existing SVG it parses the file, replaces only `layer-exits` and `layer-rooms` inner content, and writes the file back. It logs any rooms added or removed so labels can be updated manually.

---

## Room Node Rules

- **Shape:** `topLevel: true` maps → `<circle>` (outdoor). `topLevel: false` maps → `<rect rx="4">` (indoor).
- **Size:** circle `r="8"`, rect `16×16`. Both centred on the room's `(x, y)` coordinate.
- **Default fill:** `var(--bg)` — background colour covers connection lines behind the node.
- **Draw order:** exits drawn before rooms so room nodes sit on top and occlude line interiors.

**Visual states (CSS classes applied by viewer JS):**

| State | Class | Fill | Stroke |
|---|---|---|---|
| Default | `.room` | `var(--bg)` | `var(--fg)` |
| On route | `.room.route` | `#0d2d4a` | `#00d2ff` |
| Current room | `.room.current` | `#ff3b30` | `#ffffff` |
| Exit default | `.exit` | — | `#3a3a3a` |
| Exit on route | `.exit.route` | — | `#00d2ff` |

---

## Element IDs

- **Room:** `room-{identifier}` where identifier is the SHA-1 hex hash from Quow's DB.
- **Edge:** `edge-{a}-{b}` where `a` and `b` are the two room hashes sorted lexicographically. This makes the ID deterministic regardless of exit direction, and lets the viewer find an edge from any two consecutive route room IDs.
- **UU Library tile:** `room-lib-{x}-{y}` using tile coordinates (e.g. `room-lib-166-4810`).

---

## UU Library SVG (map 47)

The library has no room identifiers in the DB. Position is tracked by tile coordinates using a 30-unit grid. The SVG is generated by a dedicated `buildLibrarySvg()` function in the build script.

**Grid parameters:**
- 8 columns: x ∈ `{46, 76, 106, 136, 166, 196, 226, 256}`
- Rows: y from `4810 - N×30` to `4810 + N×30` (N configurable, default 25 → 51 rows)
- All tiles are `<rect class="room indoor">` (library is always indoors)
- Adjacent tiles connected by `<line class="exit">` (no IDs needed — library route highlighting not implemented)

**Three overlay elements** at the top of `layer-rooms`, repositioned by the viewer on `library_overlay` messages:

```svg
<line  id="lib-distortion" class="lib-distortion" visibility="hidden"/>
<circle id="lib-orb"       class="lib-orb"         visibility="hidden"/>
<path   id="lib-arrow"     class="lib-arrow"/>
```

The viewer updates their `x`/`y`/`d` attributes and `visibility` instead of canvas drawing.

## Build Script (`scripts/build-svg.mjs`)

```
node scripts/build-svg.mjs              # generate/update all SVGs
node scripts/build-svg.mjs --map 63    # single map only
```

**Algorithm per standard map:**
1. Query rooms: `SELECT identifier, x, y, short FROM rooms WHERE map_id = ?`
2. Query exits: all exits where both endpoints are on the same map; deduplicate pairs
3. Determine indoor/outdoor from `maps[mapId].topLevel`
4. If SVG exists: parse, replace `layer-exits` and `layer-rooms`, log diff
5. If SVG is new: generate all four layers, seed labels from room short names

**`package.json`** — new script:
```json
"build:svg": "node scripts/build-svg.mjs"
```

Run `npm run build:svg` after `npm run build:data` when refreshing from Quow's DB.

### mapper.js

**Map loading:**
```js
async function loadSvgMap(mapId) {
  const meta = data.maps[mapId];
  const res = await fetch(`maps/${meta.file.replace('.png', '.svg')}`);
  $mapContainer.innerHTML = await res.text();
  currentSvg = $mapContainer.querySelector('svg');
  resetZoom(mapId);   // reset viewBox width/height for new map
  centerOnRoom();     // set viewBox x/y
}
```

**State updates (CSS class toggling):**
```js
function applyState(roomId, route) {
  currentSvg.querySelectorAll('.current, .route')
             .forEach(el => el.classList.remove('current', 'route'));

  currentSvg.querySelector(`#room-${roomId}`)?.classList.add('current');

  for (const id of route) {
    currentSvg.querySelector(`#room-${id}`)?.classList.add('route');
  }
  for (let i = 0; i < route.length - 1; i++) {
    const [a, b] = [route[i], route[i+1]].sort();
    currentSvg.querySelector(`#edge-${a}-${b}`)?.classList.add('route');
  }
}
```

**ViewBox behaviour:**
- `viewBox = { x, y, w, h }` held in JS state
- **On room move (same map):** update `x` and `y` to centre on new room; `w` and `h` unchanged (zoom persists)
- **On map change:** reset `w` and `h` to default (`mapMeta.maxX / 6`); update `x`/`y` to centre on room
- **On scroll wheel:** scale `w`/`h` around pointer position (zoom)
- **On pointer drag:** shift `x`/`y` (pan); next room move will re-centre

**UU Library overlay:**
- `library_position` → remove `.current` from previous tile, add to `#room-lib-{x}-{y}`, re-centre viewBox
- `library_overlay` → update `#lib-distortion` (x/y/direction attributes + visibility), `#lib-orb` (cx/cy + visibility), `#lib-arrow` (path `d` attribute)

## Decoration Workflow

All maps ship identically from the initial build — rooms, connections, and auto-labelled names. Decoration is progressive and optional:

1. Run `npm run build:svg` — all SVG files generated with empty `layer-artwork` and seeded `layer-labels`
2. Open any SVG in Inkscape (or any SVG editor)
3. Add decorative elements inside `layer-artwork` (hull shapes, water, area fills, textures)
4. Adjust label positions in `layer-labels` for legibility
5. Use only semantic CSS classes or `var(--token)` values — no hardcoded hex colours in hand-crafted elements
6. Commit the enhanced SVG
7. Re-running `npm run build:svg` later safely updates rooms and exits without touching artwork or labels
